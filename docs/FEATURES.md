# Mint Crossing — 기능 로드맵

MVP는 "동작하는 길건너기"까지 구현됐다. 이 문서는 그 위에 붙이면 좋은 기능을
**우선순위(P0=다음, P1=곧, P2=여유될 때, P3=실험)** 와 함께 정리한다.
각 항목에 _왜_ 필요한지와 _어디_ 를 건드리면 되는지 메모를 달았다.

---

## 현재 MVP에 이미 있는 것

- 아이소메트릭 로우폴리 + 플랫 셰이딩(직교 카메라, 단일 광원 3톤)
- 큐브 캐릭터: 한 칸 전진 / 후진 / 좌우, 점프 아크 애니메이션
- 차선 절차적 생성: 잔디(나무·코인) / 도로(차) / 물(통나무 라이딩)
- 충돌 → 게임오버(차 충돌 / 익사 / 통나무 추락), 사유별 메시지
- 점수 = 최대 전진 거리, 코인 수집, 베스트 점수(localStorage)
- 입력: 키보드(↑↓←→ / WASD) + 모바일 스와이프·탭 + 화면 D패드
- 디자인 토큰(다크 네이비+민트) 그대로 재사용한 시작/HUD/게임오버 UI

---

## P0 — 다음에 바로 (게임 손맛·완성도)

1. **사운드 & 햅틱**
   - _왜_: 캐주얼 게임의 체감 품질 대부분이 점프 "톡"·코인 "칭"·충돌 "쾅"에서 온다.
   - _어디_: `src/audio/`(WebAudio 신스 또는 짧은 오픈라이선스 효과음). 엔진의
     `finishHop`/코인 획득/`die`에 콜백을 추가하거나 `EngineCallbacks` 확장.
   - iOS는 `navigator.vibrate` 미지원 → 사운드 위주, 안드로이드만 햅틱.

2. **일시정지 / 재개**
   - _왜_: 모바일에서 전화·알림으로 끊기는 건 흔하다. 백그라운드 전환 시 자동 일시정지.
   - _어디_: `GameEngine`에 `pause()/resume()`(루프에서 `dt=0` 또는 RAF 중단),
     `document.visibilitychange` 구독. HUD에 일시정지 버튼(레퍼런스 우상단 ❚❚) 복원.

3. **아이들 타임아웃(독수리/경고)**
   - _왜_: 원작 Crossy Road의 긴장감. 너무 오래 멈춰 있으면 잡아먹힌다.
   - _어디_: 엔진에 마지막 전진 타임스탬프 → 일정 시간 후 경고 셰이드 → `die('timeout')`.
     `DeathCause`에 `'timeout'` 추가.

4. **난이도 곡선 튜닝 패스**
   - _왜_: 현재 속도/밀도는 1차 추정값. 초반은 쉽고 후반은 촘촘하게 체감 조정 필요.
   - _어디_: `constants.ts`의 `SPEED_PER_ROW`, `*_BASE_SPEED`, 차선 타입 확률,
     물 연속 제한. 플레이테스트하며 숫자만 만진다.

## P1 — 곧 (리텐션·메타)

5. **캐릭터 스킨 선택** (레퍼런스 "캐릭터" 버튼)
   - _왜_: 코인 사용처 + 수집 욕구. 디자인에 이미 진입점이 있다.
   - _어디_: `buildCharacter`를 스킨 데이터(색 3종+옵션)로 파라미터화, 선택 화면 추가,
     코인으로 해금 → localStorage.

6. **코인 경제 & 일일 보상**
   - _왜_: 다시 켤 이유. 현재 코인은 표시만 되고 쓰임이 없다.
   - _어디_: 누적 코인 저장, 스킨 가격표, (선택) 광고/리워드 없이 순수 진행형.

7. **데일리 챌린지 / 시드 고정 모드**
   - _왜_: "오늘의 코스"로 친구와 같은 맵 경쟁.
   - _어디_: `Math.random`을 시드 RNG(mulberry32 등)로 교체하고 날짜를 시드로.
     엔진 생성 시 `seed?` 옵션 주입.

8. **점수 공유 카드 이미지**
   - _왜_: 자연 유입. 현재 공유는 텍스트뿐.
   - _어디_: 게임오버 시 canvas로 OG 카드 생성 → `navigator.share({files})`.

## P2 — 여유 될 때 (콘텐츠·다양성)

9. **차선 변형**: 기차(빠른 단일 위험·경보등), 강 유속 변화, 이동 장애물(굴러오는 통나무),
   미끄러운 빙판 잔디. → `LaneType` 확장 + `createLane` 분기.
10. **수집/이벤트 타일**: 자석·점프부스트·실드 파워업, 보너스 코인 무리.
11. **퍼레이드 연출**: 콤보(연속 전진) 게이지, 신기록 직전 화면 진동/글로우.
12. **랭킹**(레퍼런스 "랭킹" 버튼): 로컬 Top10 → 추후 백엔드 리더보드(Supabase 등).

## P3 — 실험 (있으면 멋진 것)

13. **세계/계절 테마**: 밤·사막·네온 등 팔레트 스왑(디자인 토큰만 교체하면 됨).
14. **리플레이/고스트**: 베스트 런 입력 기록 후 반투명 고스트와 경쟁.
15. **PWA 설치 + 오프라인**: `vite-plugin-pwa`로 홈 화면 추가·오프라인 플레이.
16. **접근성**: 색약 모드(위험을 색+패턴으로), 큰 글씨, 한손 모드 D패드 좌/우 토글.

## 기술 부채 / 정리 노트

- **번들 크기**: three.js로 인해 ~715KB(gzip 193KB). 필요 시 `three`에서 쓰는 모듈만
  임포트하거나 동적 임포트로 코드 스플릿. 지금은 게임 한 화면이라 허용 범위.
- **인스턴싱**: 차선이 많아지면 박스가 수백 개. `InstancedMesh`로 잔디/물/차 묶으면
  드로콜 급감. 모바일 저사양 대응 시 P1 승격.
- **고정 타임스텝**: 현재 가변 `dt`(상한 0.05s). 충돌 정밀도가 필요해지면 고정 스텝 누적기로.
- **엔진/React 경계**: 엔진은 프레임워크 무관(순수 TS). 콜백으로만 React에 보고하므로
  나중에 Vue/Svelte로도 재사용 가능.
